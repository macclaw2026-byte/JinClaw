# Neosgo lead engine 所需条件与工具

## 已具备（免费）
- DuckDB 本地数据库
- Python 脚本导入能力
- 去重 / 评分 / 队列 / 日报脚本
- 本地文件系统与 cron 能力

## 推荐继续使用（免费优先）
- DuckDB
- Python 3
- OpenClaw cron
- 本地 Markdown 报告

## 为了真正自动开发客户，还需要的外部条件

### 1. 发信能力（关键）
至少一个：
- Gmail / Google Workspace SMTP
- Brevo 免费层
- Resend 免费层
- Mailgun / 其他有免费额度的平台

### 2. 承接页/入口（强烈建议）
至少一个：
- trade account 申请页
- quote request 页
- catalog 下载页
- 联系表单 / WhatsApp 入口

### 3. 营销素材（关键）
- 公司介绍
- 产品范围
- 核心卖点
- 批发/项目/佣金/样品政策
- 典型案例

### 4. 反馈回流（建议）
最好至少能回写：
- sent
- bounced
- replied
- quote_requested
- won/lost

## Jacken 需要补充给系统的信息
1. 允许使用哪个发信邮箱/服务
2. 最终希望客户去哪里成交
3. Neosgo 的优先行业 / 优先州
4. 哪些客户不要联系
5. 成交定义是什么

## 不建议现在就做的事
- 没有合规与域名准备就大规模群发
- 没有回写机制就盲目自动优化
- 没有承接页就直接放大量开发流量
