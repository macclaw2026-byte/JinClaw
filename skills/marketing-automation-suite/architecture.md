# Marketing Automation Skill Suite Architecture

## Three-skill system

1. Prospect Data Engine
2. Marketing Strategy Engine
3. Outreach Feedback Engine

## Flow

- project brief enters the suite
- prospect-data-engine defines target data and builds the database
- marketing-strategy-engine turns the database into segmented strategy assets
- outreach-feedback-engine executes tasks under risk control and feeds real results back

## Why this split is reusable

Only these inputs change by project:

- business goal
- product/service
- ICP
- priority channels
- conversion target

The rest stays stable:

- evidence-first discovery
- account-based strategy generation
- controlled execution and feedback loops

## Engineering principle

- do not bind the framework to one brand
- do not assume email-first execution
- do not allow feedback to disappear outside the system
