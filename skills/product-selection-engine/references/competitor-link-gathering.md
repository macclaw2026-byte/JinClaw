# Competitor Link Gathering

Use this reference when product-selection output should include real marketplace links the user can inspect.

Use the shared monitoring standard from `../../self-cognition-orchestrator/references/monitoring-and-retrofit-framework.md`.

## Goal

For each selected or shortlisted product, provide a public marketplace link that helps the user see what the real product looks like in market.

## Preferred link order

1. direct product page
2. exact-product search results page
3. category/subcategory page with obvious relevance

## Marketplace preference

- Amazon runs -> prefer Amazon links
- Walmart runs -> prefer Walmart links
- cross-marketplace runs -> include one from each when practical

## Minimum standard

For each candidate, try to capture:
- platform
- product/sub-niche name used for search
- link type (product page | search page | category page)
- URL
- confidence (high | medium | low)

## Reliability rule

Do not invent or overstate exact product matches.

If a direct product page cannot be reliably verified from public access:
- use a stable search page instead
- or use a category page
- clearly label the link type and confidence

## Why this matters

A good research output should not only say what product to study. It should show the user where to look in the live market.
