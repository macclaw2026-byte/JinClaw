# Daily Report Cadence

Use this reference for recurring daily product-selection reporting.

## Required cadence
- Target time: **8:00 PM America/New_York**
- Frequency: daily

## Report goal
Deliver one daily report that matches Jacken's expected style:
- useful, concrete, non-repetitive
- product opportunities should not repeat from prior daily reports unless there is a meaningful new reason
- report should focus on the strongest fresh opportunities or fresh refinements

## Freshness rule
A daily report should prioritize:
1. new product opportunities not previously surfaced
2. materially refined opportunities with genuinely new evidence
3. newly discovered unmet-demand patterns that can be translated into products

## Anti-repeat rule
Do not repeat a product from a recent daily report unless at least one is true:
- new evidence materially changes confidence
- a new marketplace/platform fit appears
- a new product-improvement path makes it meaningfully different
- a previous idea is being split into a more specific sub-product that is genuinely new

## Fallback rule
If fresh opportunities are weak on a given day:
- prefer fewer stronger new ideas over padding with repeats
- if needed, report adjacent but non-duplicate sub-niches, product improvements, or unmet-demand translations
