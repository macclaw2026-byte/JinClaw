# Niche Extension Policy

Use the shared monitoring standard from `../../self-cognition-orchestrator/references/monitoring-and-retrofit-framework.md` when deciding whether a niche deserves its own extension.

Keep `us-market-growth-engine` general.

When a niche repeatedly appears, create a focused extension skill instead of overloading the core.

## Good extension candidates

- US lighting B2B growth
- furniture buyer outreach
- interior designer acquisition
- contractor lead development
- local commercial category prospecting

## Rule

The core skill should teach the reusable marketing framework.
The niche extension should teach the vertical-specific buyer logic, message angles, channels, and conversion tactics.

## Extension trigger checks

Create or improve an extension when:
- the same niche recurs
- the same monitoring gaps recur
- buyer/channel/conversion logic clearly diverges from the generic model
